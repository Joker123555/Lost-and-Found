# 校园失物招领系统

基于微信小程序的校园失物招领平台，帮助师生快速发布、匹配失物与招领信息，内置智能匹配与在线沟通功能。

## 📱 项目简介

本系统采用前后端分离架构，面向校园场景提供便捷的失物招领服务。  
- **用户端**：微信小程序，支持一键登录、发布信息、智能匹配、在线沟通等。  
- **管理端**：Web 后台管理系统，支持信息审核、用户管理、数据统计等。

项目通过智能匹配算法自动推荐相似物品，提高找回率；内置聊天功能保障隐私，避免直接泄露联系方式。

## 📁 主要模块说明

### 小程序端核心页面

- **首页**：展示所有已发布的寻物/招领信息，支持切换类型、下拉刷新、上拉加载。
- **智能匹配**：基于用户发布历史，推荐匹配度高于阈值的物品，按相似度排序。
- **消息**：会话列表，支持文字聊天（后续可扩展图片）。
- **我的**：个人中心，管理发布记录、修改密码、查看公告、意见反馈。

### 管理端核心模块

- **信息审核**：待审核列表，支持详情查看、通过/驳回（驳回需填写原因）。
- **用户管理**：封禁/解封用户，封禁后用户无法登录且所有发布自动下架。
- **数据统计**：使用 ECharts 展示发布趋势、分类占比。
- **公告管理**：富文本编辑器发布公告，用户端可见。
- **反馈管理**：查看用户反馈并标记处理状态。

## 🗃 数据库设计简表

| 表名 | 说明 |
|------|------|
| `users` | 用户表（支持微信 openid 和手机号登录，角色区分普通用户/管理员） |
| `items` | 物品表（type 区分寻物/招领，status 管理审核与认领状态） |
| `item_images` | 物品图片表 |
| `claims` | 认领/归还记录表（追踪认领流程） |
| `matches` | 智能匹配记录表（存储匹配结果及匹配度） |
| `chat_sessions` | 聊天会话表 |
| `chat_messages` | 聊天消息表 |
| `announcements` | 公告表 |
| `feedbacks` | 反馈表 |

详细字段请参考项目文档 `docs/数据库设计.md`。
